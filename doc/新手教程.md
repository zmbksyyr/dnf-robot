# DNF Robot 新手图文教程

## 1. 系统概述

本系统用于管理 DNF 私服的假人机器人，由两部分组成：
- **robot 假人程序**：运行在 CentOS VM 上，管理假人上线/移动/喊话/摆摊
- **部署启动器**（`部署.exe`）：Windows GUI 工具，一键部署/重启 robot 并启动游戏服务

VM 连接信息：`192.168.200.131` / SSH `root:123456` / Web 密码 `twadmin`

---

## 2. 虚拟机准备

### 2.1 启动虚拟机

确保 VMware 中的 CentOS VM 已启动，可以 ping 通 `192.168.200.131`。

### 2.2 准备部署文件

将 `部署.exe` 和 `robot` 放在同一个文件夹中：

```
D:\deploy\
  ├── 部署.exe      ← 部署启动器（双击运行）
  └── robot         ← 假人控制程序（编译产物）
```

### 三个按钮功能

| 按钮 | 做了什么 | 适用场景 |
|------|---------|---------|
| **部署 robot** | 上传 → 校验 → 备份 → 杀旧 → 替换 → 启动 | 更新了 robot 程序，或首次部署 |
| **重启 robot** | 杀旧 → 启动（不上传不替换） | robot 卡住/异常，快速重启 |
| **启动 /root/run** | 后台执行游戏启动脚本，监控进程和端口 | 游戏服务挂了、VM 重启后 |

## 4. 首次部署流程

### 步骤 1：部署 robot

点 `[部署 robot]`，日志区逐步输出：

```
[08:00:01] 本地 robot: D:\deploy\robot (11243682 bytes)
[08:00:01] SSH 192.168.200.131 连接成功
[08:00:02] 上传 /root/robot.new 完成
[08:00:02] 文件大小校验通过 (11243682 bytes)
[08:00:02] chmod +x 完成
[08:00:02] 备份旧 robot → /root/robot.bak.20260625_080002
[08:00:03] 旧 robot 已停止
[08:00:03] 替换 robot 完成
[08:00:05] 新 robot 已启动 (pid: 2719)
```

弹窗 "部署成功" 即完成。

### 步骤 2：启动游戏服务

点 `[启动 /root/run]`：

```
[08:00:06] SSH 连接成功，正在启动游戏服务 ...
[08:00:06] /root/run 已提交，开始监控 ...
[08:00:12] df_game_r 进程已稳定 (2 个)，等待端口就绪 ...
[08:00:18] 游戏端口 10011 已监听
[08:00:20] 游戏端口 10056 已监听
```

弹窗 "启动成功" 即游戏服务已就绪。

### 步骤 3：打开 Web 管理

浏览器访问 `http://192.168.200.131:8112`，输入密码 `twadmin`。

### 步骤 4：开启自动调度

- Auto 面板输入目标人数（如 `20`）
- 点 `Start`
- 假人开始自动上线、移动、喊话、摆摊

### 顶部状态卡片

| 卡片 | 说明 | 异常时 |
|------|------|--------|
| 🤖 Robot | robot 进程状态 | 重启 robot |
| 🗄️ Database | MySQL 连接 | 检查 MySQL |
| 🔑 Key | 加密密钥 | 点"释放默认 key" |
| 🎮 Game Port | 游戏端口 | 启动 /root/run |
| 📡 Monitor | 喊话服务 | 启动 /root/run |

所有卡片绿色才能正常操作。

### 核心操作

- **Auto**：设置目标在线人数，点 Start 开启自动调度
- **Online**：手动让指定假人上线
- **Move**：让选中假人移动
- **Shout**：普通喊话
- **World Shout**：全服喊话
- **Store**：让假人摆摊
- **Logout**：让假人下线（不删除数据）
- **Cleanup**：清理损坏的假人数据
- **Config**：修改行为配置

## 7. 注意事项

1. 首次部署前确保 VM 已启动，能 ping 通 `192.168.200.131`
2. Web 管理页面密码默认为 `twadmin`
3. 部署 robot 自动保留最近 3 个备份 (`/root/robot.bak.*`)
4. Cleanup 只用于清理损坏 UID，不要替代正常下线
5. 自动模式下，手动操作未接管 UID 会返回 `uid_not_attached`
6. VM CPU 长时间异常时检查 `vmtoolsd`（正常应 < 3%）
7. 遇到奇怪问题，先点 `[重启 robot]` 试试
8. 不要手工修改 `[system]` 段配置

## 8. 常见问题

**Q: 部署后假人不上线？**
A: 点 `启动 /root/run` 等端口就绪 → Web 中点 Auto Start。

**Q: 网页打不开？**
A: SSH 执行 `pgrep -af robot` 查进程。没活就点 `重启 robot`。

**Q: 部署时提示旧进程未停止？**
A: 不影响，程序会警告后继续。僵尸进程需重启 VM 清理。

**Q: 端口数和进程数不一致？**
A: 正常。有的 df_game_r 是内部通信进程，不绑定外部端口。

**Q: 日志大量 `online_confirm_timeout`？**
A: 目标人数太高，游戏服处理不过来。降低目标人数。

**Q: 三个按钮都灰了点不了？**
A: 上一操作还在执行中，等日志区显示完成或弹窗出现。
