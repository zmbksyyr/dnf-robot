# 使用说明

## 部署

1. 上传 `robot` 到服务器 `/root/robot`。
2. 执行 `chmod +x /root/robot`。
3. 启动：`nohup /root/robot >/dev/null 2>&1 &`。
4. 打开 Web：`http://服务器IP:8112`。
5. 输入 Web 密码，默认 `twadmin`。

robot 固定使用：

- 主配置：`/root/config/config.ini`
- 行为配置：`/root/config/robot_config.ini`
- 主日志：`/root/config/log_robot`

## 第一次使用

1. 启动游戏服务，确认 `10011` 端口开放。
2. 打开 Web，查看 Robot、游戏端口、Key 状态。
3. Key 状态为 `用户` 或 `默认` 时可以操作。
4. Key 状态为 `缺失` 或 `无效` 时，先自己配置 game 目录密钥，或点击 `释放默认 Key`。
5. game 使用默认 key 时，Web 提供下载密钥对按钮，用于放到游戏客户端目录。

## Web 操作

- 自动行为：设置目标在线人数并启动/停止自动调度。
- 上线：让指定假人上线并保活。
- 移动、喊话、摆摊：对指定假人执行一次动作。
- 下线：真实下线并结束 actor。
- 配置：修改行为配置；底层 `[system]` 不在 Web 修改。
- 日志：查看 `/root/config/log_robot`。

## 关键配置

- `[spawn]`：假人出生点和活动范围。`spawn_fixed=false` 时优先使用 `follow_account` 附近或角色原位置；`spawn_fixed=true` 时使用配置的城镇、区域、坐标范围。
- `[auto]`：自动目标人数、移动/喊话/摆摊间隔、摆摊概率。
- `[scheduler]`：UID 回收阈值、指标间隔、摆摊并发、上线启动速率。
- `[system]`：actor 轮询、手动动作超时、发包节奏。通常不要改。

## 默认行为

- 默认目标在线人数：`20`。
- 每个假人独立随机移动、喊话、材料摆摊。
- 摆摊期间暂停移动，喊话照常。
- 摆摊只出售已验证的材料，不出售装备。
- 自动关闭会释放自动 actor 并下线当前租约；手动操作未被自动 actor 租用的 UID 时，直接走基础能力，不创建额外 actor。

## 注意

- 不要手工修改底层 `[system]`，除非明确知道影响。
- 只有一个主日志，不记录喊话内容。
- 页面异常时先刷新页面，再查看主日志。
