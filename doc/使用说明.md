# 使用说明

## 部署

推荐用 `/root/run` 启动全服服务。

1. 上传 `robot` 到服务器 `/root/robot`。
2. 执行 `chmod +x /root/robot`。
3. 启动全服：`/root/run`。
4. 打开 Web：`http://服务器IP:8112`。
5. 输入 Web 密码，默认 `twadmin`。

robot 固定使用：

- 主配置：`/root/config/config.ini`
- 行为配置：`/root/config/robot_config.ini`
- 主日志：`/root/config/log_robot`
- 启动输出和 Web 诊断日志：`/root/robot_stdout.log`

## 第一次使用

1. 启动游戏服务，确认 `10011` 端口开放。
2. 确认 monitor 服务可用，全服喊话依赖 `127.0.0.1:30303`。
3. 打开 Web，查看 Robot、Game Port、Monitor、Database、Key 状态。
4. Key 状态为 `User Key` 或 `Project Default Key` 时可以操作。
5. Key 状态为 `Missing Key`、`Invalid Key` 或 `Incomplete` 时，先配置 game 目录密钥，或点击释放默认 key。
6. 使用默认 key 时，Web 提供下载默认 key 按钮，用于放到游戏客户端目录。

## Web 操作

- Auto：设置目标在线人数并启动/停止自动调度。
- Online：让指定假人上线并保活。
- Move：对指定假人执行一次移动。
- Shout：执行普通喊话。
- World Shout：执行全服喊话。
- Store：对指定假人摆摊。
- Logout：让指定假人下线，但不删除数据库数据。
- Cleanup：删除假人 registry/database 相关行，只用于坏 UID、明确选中 UID 或清理命令。
- Config：修改行为配置；底层 `[system]` 通常不要改。
- Log：查看 `/root/config/log_robot`。

## 自动与手动

自动开启时：

- 调度维护目标在线人数。
- actor 数不足目标时自动补 actor。
- idle actor 会自动装入 UID。
- auto actor 会自动移动、喊话、摆摊。
- 用户只能操作已经被 actor 接管的 UID；未接管 UID 返回 `uid_not_attached`。

自动关闭时：

- 调度进入 `manual`。
- 自动 actor 会被停止并下线。
- 不再自动补人、分配 UID、移动、喊话、摆摊。
- 用户手动操作未接管 UID 时，系统会先创建必要 actor 并装入该 UID，再执行动作。

## 关键配置

- `[spawn]`：假人出生点和活动范围。`spawn_fixed=false` 时优先使用 follow account 附近或角色原位置；`spawn_fixed=true` 时使用配置的城镇、区域、坐标范围。
- `[auto]`：自动目标人数、移动/喊话/摆摊间隔、摆摊概率。
- `[scheduler]`：UID 回收阈值、指标间隔、摆摊并发、上线启动速率、breaker/端口异常释放批次。
- `[system]`：actor 轮询、手动动作超时、发包节奏。通常不要改。

## 默认行为

- 默认目标在线人数：`20`。
- 每个假人由独立 actor 接管。
- actor 会按状态独立移动、喊话、材料摆摊。
- 摆摊期间暂停移动，喊话仍可执行。
- 摆摊只出售已验证的材料，不出售装备。
- 全服喊话通过 monitor 发送，不依赖 `.so hook`。
- 缩容、停止自动、Logout 都不 cleanup 正常 UID。

## 清理规则

- Cleanup 只用于假人数据，不用于正常账号。
- cleanup 候选必须能通过 registry、accountname、Dummylist 等信息证明是假人。
- accountname 不等于 UID 字符串的记录会被保护。
- registry account 不等于 UID 字符串的记录会被保护。
- 非内部 broken cleanup 缺少 Dummylist 行会被保护。
- Cleanup 执行前会先停止相关 UID，避免边在线边删数据。

## 排障入口

- 页面异常、变红、回登录页：先看 `/root/robot_stdout.log`。
- 业务动作失败：看 `/root/config/log_robot`。
- 调度不补人：看 Web 的 scheduler mode/reason、Key、Game Port、Monitor、Database。
- 用户命令返回 `scheduler_busy`：说明正在结构性操作、actor 容器变更、自动 fill/loading 或 actor 状态不稳定。
- 用户命令返回 `uid_not_attached`：自动模式下该 UID 未被 actor 接管。

## 注意

- 不要手工改底层 `[system]`，除非明确知道影响。
- 不要用 cleanup 代替正常下线或缩容。
- 不要手工删除正常数据库表来清假人。
- 不需要部署 `.so hook`。
- 不需要 patch `df_game_r`。
