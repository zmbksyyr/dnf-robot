# 随机稳定性压测脚本调用说明

脚本路径：

```bash
/root/vm_random_stability.py
```

仓库路径：

```bash
tools/vm_random_stability.py
```

## 默认运行

```bash
python /root/vm_random_stability.py
```

默认参数：

```text
--hours 1.0
--sample-interval 10
--log-snapshot-interval 300
--target-min 100
--target-max 600
```

脚本只保留全测模式；`--hours` 控制全测总时长，所有场景节点按比例压缩或拉伸。

脚本包含破坏性场景，会自动备份、自动恢复、自动生成报告。

## 后台运行

```bash
nohup python /root/vm_random_stability.py > /root/vm_random_stability.out 2>&1 &
```

## 常用调用

指定运行时长：

```bash
python /root/vm_random_stability.py --hours 3
```

长测：

```bash
python /root/vm_random_stability.py --hours 8
```

降低目标人数范围：

```bash
python /root/vm_random_stability.py --target-min 100 --target-max 400
```

关闭 cleanup：

```bash
python /root/vm_random_stability.py --no-cleanup
```

禁止 cleanup 在线 UID：

```bash
python /root/vm_random_stability.py --no-allow-online-cleanup
```

指定输出目录：

```bash
python /root/vm_random_stability.py --out-dir /root/robot_stability_manual
```

固定随机种子：

```bash
python /root/vm_random_stability.py --seed 123456
```

## 查看结果

查看最新输出目录：

```bash
ls -td /root/robot_stability_* | head -n 1
```

查看报告：

```bash
cat /root/robot_stability_*/report.md
```

查看失败项：

```bash
cat /root/robot_stability_*/failures.json
```

查看事件日志：

```bash
tail -f /root/robot_stability_*/events.log
```

查看采样 CSV：

```bash
tail -n 20 /root/robot_stability_*/samples.csv
```

查看日志快照：

```bash
tail -n 200 /root/robot_stability_*/collected_logs.log
```

查看场景快照：

```bash
ls /root/robot_stability_*/snapshots
```

## 输出文件

```text
events.log
samples.csv
collected_logs.log
summary.json
report.md
failures.json
snapshots/
baseline/
```

`snapshots/` 保存每个场景前后的系统快照，包括 API 状态、端口、进程、关键文件 hash、DB 统计、DB 表结构、TCP 状态和磁盘状态。

`baseline/` 在测试开始时生成，包含配置、iteminfo 和市场机器人上架记录备份。

脚本正常结束、异常退出或 Ctrl+C 中断时，会自动执行最终恢复：

```text
恢复 baseline
执行 /root/run
重启 /root/robot
等待 robot API
启动 market auto
等待双拍卖服务恢复
采集 final 日志
```

手动恢复基线只作为兜底：

```bash
sh /root/robot_stability_*/baseline/restore_baseline.sh
```
